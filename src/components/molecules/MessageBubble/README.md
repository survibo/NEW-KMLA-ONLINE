# MessageBubble

`MessageBubble`은 범용 채팅 UI 기본 컴포넌트가 아니라, 메시지 페이지 전용 프로덕트 컴포넌트입니다.

## 목적

- DM 스레드 UI에서 하나의 시각적 메시지 단위를 렌더링합니다.
- 현재의 시각적 방향을 유지하면서 메시지 페이지 규칙을 명확히 합니다.
- 향후 메시지 스레드 래퍼 및 페이지 작업의 기준으로 활용합니다.

## 범위

`MessageBubble`은 프레젠테이션을 담당합니다.

`MessageBubble`이 결정하지 않아야 할 것:

- 실시간 상태
- Supabase 구독
- 읽음 확인 계산
- 이전/다음 메시지로부터의 그룹 계산
- 스레드 내 어느 메시지에 아바타, 이름, 시간, 상태를 표시할지

이 결정들은 추후 스레드 레벨 래퍼가 담당합니다.

## 현재 Props

### 콘텐츠

- `content`
- `image`
- `file`
- `replyTo`

### 상태

- `isOwn`
- `isEdited`
- `isDeleted`
- `metaLabel`

### 그룹핑 / 표시

- `groupPosition`: `single | start | middle | end`
- `showAvatar`
- `showName`
- `showTime`
- `time`
- `timeFormat`

## 렌더링 규칙

### 그룹 형태

- `single`: 독립 버블
- `start`: 벽쪽 하단 모서리가 직각이 됨
- `middle`: 벽쪽 상단 및 하단 모서리가 직각이 됨
- `end`: 벽쪽 상단 모서리가 직각이 됨

벽쪽(wall-side)의 의미:

- 수신 메시지는 왼쪽
- 발신 메시지는 오른쪽

### 그룹 간격

- 그룹화된 `middle` 및 `end` 버블은 이전 버블과 약간 더 가깝게 배치됩니다.
- 이는 시각적 처리이며, 그룹 계산은 여전히 호출자가 제어합니다.

### 답장 미리보기

- 답장 미리보기는 실제 답장 버블 뒤에 위치합니다.
- 미리보기 박스는 활성 버블보다 더 흐리게 표시됩니다.

### 수정된 메시지

- `isEdited`가 true이면 `수정됨`을 표시합니다.
- 수정됨 라벨은 버블 본문 위에 표시됩니다.
- 삭제된 메시지에는 수정됨 라벨을 표시하지 않습니다.

### 삭제된 메시지

- `isDeleted`가 true이면 본문을 `메시지가 삭제되었습니다`로 대체합니다.
- 삭제된 메시지에는 답장 미리보기, 이미지, 파일 첨부를 렌더링하지 않습니다.
- 삭제된 메시지는 흐린 중립 톤의 버블을 사용합니다.

### 메타 라벨

- `metaLabel`은 발신 트레일링 상태 전용입니다.
- `isOwn === true`일 때만 표시됩니다.
- `groupPosition === 'single'` 또는 `groupPosition === 'end'`일 때만 표시됩니다.
- `전송됨`, `읽음`

## 알려진 제한 사항

현재 의도적인 제한이며, 추후 래퍼에서 처리되어야 합니다.

- `groupPosition`, `showAvatar`, `showName`, `showTime`은 여전히 개별적으로 전달됩니다.
- 삭제됨 및 답장 라벨은 프로덕트 텍스트이며, 아직 외부화된 카피가 아닙니다.
- 일부 답장/그룹 겹침은 전용 레이아웃 모델 대신 시각적 튜닝 값을 사용합니다.

## 다음 단계

다음 메시지 페이지 리팩토링에서는 아래를 수행하는 스레드 레벨 래퍼를 도입해야 합니다:

- 메시지 도메인 데이터를 수신합니다.
- `groupPosition`을 계산합니다.
- 아바타/이름/시간/상태 표시 여부를 결정합니다.
- 프로덕트 메시지 상태를 `MessageBubble` props로 매핑합니다.

그 전까지는 `MessageBubble`을 메시지 페이지 전용 프로덕트 렌더링 컴포넌트로 취급하세요.
